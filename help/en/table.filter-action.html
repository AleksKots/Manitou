7.  Table: filter_action

The actions associated to conditions of the filtering system.

Structure of filter_action

expr_id

integer

Expression to which the action is connected.

REFERENCES filter_expr

action_order

smallint

Order of the action in the list of actions connected to the expression, starting at 1.

action_arg

text

A string that qualifies the action and whose interpretation depends on action_type. Possible values are:

  • if action_type='tag', the tag's name flattened with the substring '->' as a node separator for hierarchies.

  • if action_type='priority', '+=x' as the action arg will add x (a signed integer) to the current priority of the message, and '=x' will set the current priority to x.

  • if action_type='status', the action arg is a substring containing letters 'R' for read, 'A' for archived, 'T' for trashed and 'D' for discard.

action_type

character varying(100)

The kind of the action. Possible values are: priority, redirect, status, stop, tag.
